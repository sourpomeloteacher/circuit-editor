# 回路図エディタ（工業高校電気科教員向け）

## 今回の修正内容

- **選択が保持されない不具合を修正**: 部品グループの`click`イベントがキャンバス背景までバブリングし、選択直後に解除されていたのが原因でした。部品グループに`onClick`での伝播停止を追加し、選択が保持されるようにしました。これで右パネルの数値・単位・問題モードの編集がクリック後も継続してできます。
- **抵抗記号を現行JIS/IEC規格（長方形）に変更**: 旧来のジグザグ記号を廃止しました。
- **未実装だった部品記号を追加**: 三相交流電源・電池・切換スイッチ・ブリッジダイオード・NPN/PNPトランジスタ・FET・リレーコイル・a/b/c接点をすべて実装しました。

## 現在の実装状況

- [x] フェーズ0〜3: 基盤・データモデル・基本キャンバス・編集機能
- [x] フェーズ4: 配線機能（ピンをクリックして配線、90°/自由配線切替、接続点表示、配線クリックで削除）、複数選択（Shift+クリック）、コピー＆ペースト（Ctrl+C/V）
- [x] フェーズ5: 部品ライブラリ全種の記号実装（JIS準拠の抵抗記号含む）
- [x] フェーズ6: ラベル機能（部品番号の自動採番・振り直し、表示/非表示切替）
- [x] フェーズ7: テンプレート15種（オームの法則〜三相Δ結線）をワンクリック配置
- [x] フェーズ8: テスト作成支援（問題モード一括切替、一括数値変更、ランダム数値生成、問題A/B/C自動生成、部品番号振り直し）
- [x] フェーズ9: 出力機能（PNG/SVG/PDF保存、クリップボードへ画像コピー、印刷用CSS）

## 既知の制約（今後の改善候補）

- ドラッグでの範囲選択（矩形選択）は未実装。複数選択は現状Shift+クリックのみ対応。
- コピー＆ペーストは部品のみが対象で、部品間の配線はコピーされません。
- 配線の経路はピン位置から自動計算する方式のため、手動での中間点編集はできません（部品を動かせば自動追従します）。
- PDF出力は画面表示をラスタライズして埋め込む方式です（ベクターPDFではありません）。将来的にSVG→PDFのベクター変換ライブラリへの置き換えを検討してください。
- キーボードショートカットはUndo/Redo(Ctrl+Z/Y)、コピー/貼付け(Ctrl+C/V)、削除(Delete)、配線キャンセル(Esc)のみ対応。

## 新機能の使い方（フェーズ4〜9）

- **配線**: 部品の端子（薄い丸印）をクリック→別の部品の端子をクリックすると配線されます。配線中にキャンバスの何もない場所をクリックするかEscキーでキャンセルできます。配線自体をクリックすると削除されます。ツールバーの「配線」プルダウンで90°配線／自由配線を切り替えられます。
- **複数選択**: Shiftキーを押しながら部品をクリックすると選択に追加/除外できます。まとめて回転・削除・コピーが可能です。
- **コピー＆ペースト**: 選択後にツールバー/右パネルの「コピー」、またはCtrl+Cで複製、Ctrl+Vまたは右パネルの操作で貼り付けます。
- **テンプレート**: 左パネル下部の「テンプレート」から回路名をクリックすると、その場に回路一式が配置されます。
- **テスト作成支援**（右パネル下部）:
  - 「問題モード」チェックで全部品の数値が□表示になります（解答用は再度チェックを外す）
  - 「一括数値変更」で倍率／加算による一括変更ができます
  - 「数値をランダム生成」で対応部品の数値をランダムに振り直します
  - 「A/B/Cを生成」で同じ回路構造のまま数値だけ異なる3パターンを作成し、タブで切り替えて個別に保存できます
  - 「部品番号を振り直す」で配置順にR1, R2...のような番号を再採番します
- **出力**: ツールバーのPNG保存/SVG保存/PDF保存/画像コピー/印刷。Word・PowerPointへは「画像コピー」→貼り付け、または「PNG保存」した画像の挿入をおすすめします。

## 動作確認・ビルド手順（お手元のWindows PCで実行してください）

このプロジェクトはオフライン環境で作成されているため、依存パッケージのダウンロードとビルドはお手元の環境（インターネット接続あり）で行ってください。

### 1. 必要なソフトウェアのインストール

1. **Node.js 20以上**: https://nodejs.org/ からLTS版をインストール
2. **Rust**: https://www.rust-lang.org/tools/install から `rustup-init.exe` を実行してインストール
3. **Visual Studio C++ Build Tools**: Tauriのビルドに必要です。以下を実行してインストールを促されます（または https://visualstudio.microsoft.com/visual-cpp-build-tools/ から「C++によるデスクトップ開発」ワークロードを選択してインストール）
4. **WebView2 Runtime**: Windows 11には標準搭載されているため通常は追加作業不要です

### 2. 依存パッケージのインストール

プロジェクトフォルダ（`circuit-editor`）をコマンドプロンプトまたはPowerShellで開き、以下を実行してください。

```powershell
npm install
```

### 3. アイコンの生成（初回のみ）

`src-tauri/tauri.conf.json` はアイコンファイル（`src-tauri/icons/`）を参照しますが、このプロジェクトには画像ファイルを含めていません。お好きな正方形のPNG画像（1024x1024推奨）を用意し、以下を実行してアイコン一式を自動生成してください。

```powershell
npx tauri icon path/to/your-logo.png
```

画像が用意できない場合は、`src-tauri/tauri.conf.json` の `bundle.icon` の行を一時的に削除しても動作確認は可能です（配布用ビルド時には推奨しません）。

### 4. 開発モードで起動（動作確認）

```powershell
npm run tauri dev
```

初回はRustのビルドに数分かかります。ウィンドウが立ち上がれば成功です。

### 5. 配布用 .exe のビルド

```powershell
npm run tauri build
```

成功すると、以下にインストーラが生成されます。

```
src-tauri/target/release/bundle/nsis/回路図エディタ_0.1.0_x64-setup.exe
```

このインストーラを配布することで、他の先生のPCにもインストールして利用できます（Windows 11標準のWebView2を利用するため、追加ランタイムのインストールは通常不要です）。

### トラブルシューティング

- `cargo` コマンドが見つからない → Rustインストール後にターミナルを再起動してください
- ビルド時に `link.exe` が見つからないエラー → C++ Build Toolsのインストールを確認してください
- 日本語ファイル名で保存できない → Windowsの「ベータ: ワールドワイド言語サポートでUnicode UTF-8を使用」設定を有効にすると解消する場合があります

## アーキテクチャ上の注意（今後の実装を引き継ぐ場合）

- **Core層（`src/core/`）にはUIやTauri固有のコードを絶対に書かない**こと。将来のWeb版移植時にこの層はそのまま流用する前提。
- 新しい編集操作を追加する場合は `src/core/commands/` に `Command` インターフェースを実装したクラスを追加し、`useCircuitStore` から `HistoryManager.applyCommand()` 経由で呼び出すこと。これによりUndo/Redoが自動的に効く。
- ファイル保存やクリップボードなどOS依存の処理は必ず `PlatformAdapter` 経由で行い、`TauriAdapter` 以外の場所から直接Tauri APIをimportしないこと。
